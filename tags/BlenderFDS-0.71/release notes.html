BlenderFDS release notes

Have you ever passed long hours calculating and typing geometric coordinates of FDS objects by hand? That is a really slow, tedious and error prone procedure. BlenderFDS is the open source graphical interface for FDS that assists the user during the input file preparation.

BlenderFDS efforts are completely unrelated and independent from the FDS development performed at NIST and other organizations. None of these organizations finance, support, endorse, or otherwise recommend BlenderFDS.

  • BlenderFDS main site: http://www.blenderfds.org/

  • Consult user documentation at: http://code.google.com/p/blenderfds/wiki/Home?tm=6

  • Howto install at: http://code.google.com/p/blenderfds/wiki/InstallingBlenderFDS

  • Subscribe to the discussion group: http://groups.google.com/group/blenderfds

  • Report an issue at: http://code.google.com/p/blenderfds/issues/list

2011-03-09: BlenderFDS 0.71 “Ku-Ring-Gai”

Blender 0.71 must be installed on Blender 2.56a beta. It will not work with other versions.

This release fixes some minor 0.70 bugs discovered early in the morning by passionate users:

  • do not insert an ID parameter in several namelists as SLCF, HOLE... (see this FDS issue).

  • UI error message on non-manifold voxelized objects is not displayed in edit mode.

  • Improve MESH comments.

  • FDS input files should never exceed 100 colums, really!

2011-03-08: BlenderFDS 0.70

Blender 0.70 must be installed on Blender 2.56a beta. It will not work with other versions.

Blender API has changed again. BlenderFDS has evolved still. Its code has been largely refactored for speed and maintainability. Some aspects of the user interface have been improved. Many small bugs detected by users were solved.

The annoying “UI freezes when exporting” issue is still not solved: Blender fault. When you export a complex file, you shall wait several minutes for the process to complete. During the process the user interface remains unresponsive...

BlenderFDS file format has changed slightly. BlenderFDS can import and update your old files created with previous versions: geometry is retained, but double check the properties just to be sure that everything works well.
